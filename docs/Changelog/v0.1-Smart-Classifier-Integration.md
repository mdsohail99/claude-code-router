
# Changelog 0.1 — Smart Classifier & Stealth Integration (March 2026)

This changelog documents the complete overhaul of the Claude Code Router (CCR) to introduce native task classification, vision support, and privacy-first CLI configurations.

---

## 🛠️ Changes & Improvements

### 1. 🧠 Native Smart Task Classifier
- **Scope**: `packages/core/src/utils/router.ts`
- **Logic**: Integrated a Tool-Aware decision tree into the core `getUseModel()` function.
- **Goal**: Automatically switch to high-performance models for coding and image tasks without manual switching or agent-overwrites.
- **Detected Tools**: `str_replace_editor`, `bash`, `read_file`, `ls`, `glob`, `grep_search`.

### 2. 🖼️ Native Vision Support
- **Scope**: `packages/core/src/utils/router.ts`
- **Logic**: Added native image-intent detection in the message stream.
- **Goal**: Fixes the "Image Race Condition" where agent-led model switches were overwritten by the core router. CCR now prioritizes vision models when an image payload is detected.

### 3. 🛡️ Privacy & Stealth CLI
- **Scope**: `packages/cli/src/utils/createEnvVariables.ts`
- **Logic**: Hardcoded the suppression of attribution and telemetry.
- **Values**: 
  - `CLAUDE_CODE_ATTRIBUTION_HEADER: "0"`
  - `DISABLE_TELEMETRY: "1"`

### 5. 🔄 Interactive Model Selector & Selector Loop
- **Scope**: `packages/cli/src/utils/modelSelector.ts`
- **Logic**: Converted the `ccr model` selector into a persistent `while` loop.
- **Goal**: Allows users to make multiple category changes (Default, Background, Image, etc.) and then manually trigger a **"🚀 Apply & Restart Service"** action.
- **Sync**: This final action synchronizes the `Default Model` into Claude Code's native `settings.json` and restarts the CCR background service.

## 🛠️ Relocation Update
The core routing matrix and PowerShell automation suite have been moved from the `HeliSync` repository to the root of the CCR repository:
- `scripts/ccr/ccr-route-matrix.json` (Now local to CCR)
- `scripts/ccr/*.ps1` (Automation suite)

### Claim 1: `code` Category was Dead Code
- **Verified ✅**: The official `getUseModel` in `router.ts` had no checks for coding tools. `'code'` wasn't even a valid `RouterScenarioType`.
- **Patched**: It is now a first-class routing category with native priority.

### Claim 2: `image` Routing Race Condition
- **Verified ✅**: The `ImageAgent` was trying to mutate the model *after* the Core Router had already made its fallback decision to `default`. 
- **Patched**: Routing is now centralized in `core`, eliminating the "fighting" between agents and the engine.

### Claim 3: Attribution Suppression
- **Verified ✅**: `CLAUDE_CODE_ATTRIBUTION_HEADER` was entirely absent from the original CLI.
- **Patched**: It is now explicitly suppressed to provide a stealthy experience.
